# 🧠 系统设计文档：年薪百万作战系统

## 第一性原理推导

### 为什么人会半途而废？

1. **意志力有限** → 解决方案：自动化减少决策消耗
2. **反馈延迟** → 解决方案：每日可视化进度+AI即时反馈
3. **环境摩擦** → 解决方案：利用通勤/午休碎片时间，周末整块时间
4. **目标模糊** → 解决方案：拆解到每日可执行动作
5. **孤独作战** → 解决方案：公开承诺+网络监督
6. **家庭冲突** → 解决方案：早起+碎片时间，绝不占用陪娃时间

### 核心约束（不可违背）

| 约束 | 说明 | 应对策略 |
|-----|------|---------|
| 21:00下班 | 晚上无整块时间 | 通勤+午休成为主战场 |
| 22:00到家 | 陪娃到23:00 | 早起1小时+周末整块 |
| 周末陪娃 | 无法全天学习 | 早起2h+娃午睡2h+晚上1h |
| 每周19h | 时间减半 | 提高密度，减少废动作 |

### 系统架构

```
┌─────────────────────────────────────────┐
│           用户（你）                    │
│  输入：每日日志、算法提交、代码提交      │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         GitHub 仓库（中央数据库）         │
│  - daily/：每日日志                      │
│  - algorithm/：刷题进度                  │
│  - projects/：项目文档                   │
└──────────────┬──────────────────────────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌───▼───┐  ┌──▼───┐  ┌──▼───┐
│GitHub │  │  AI  │  │ 网络 │
│Actions│  │Review│  │监督  │
│       │  │      │  │      │
│- 每日 │  │- 每周│  │- 公开│
│  检查 │  │  复盘│  │  承诺│
│- 每周 │  │- 优化│  │- 朋友│
│  报告 │  │  建议│  │  监督│
└───────┘  └───────┘  └───────┘
```

### 核心机制

| 机制 | 原理 | 实现 |
|-----|------|------|
| **自动化检查** | 减少意志力消耗 | GitHub Actions定时检查 |
| **AI复盘** | 即时反馈+客观诊断 | 每周调用AI API分析日志 |
| **公开承诺** | 社会压力驱动 | 仓库公开+朋友监督+违约惩罚 |
| **可视化进度** | 游戏化反馈 | README动态仪表盘 |
| **模板化日志** | 降低记录摩擦 | 每日模板，填空式记录 |
